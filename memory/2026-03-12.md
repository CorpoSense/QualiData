### IMPORTANT: DRY Principle for Shared Functions
**Problem:** We had duplicate `save_operation` functions in both `operations.py` and `operations_extra.py`. This caused:
- Inconsistent implementations
- Bugs in one but not the other
- Harder maintenance

**Solution:** 
- Created single `save_operation` in `app/utils/operations.py`
- All routers now import from `app.utils.operations`
- DRY principle: define once, use everywhere

**Best Practices:**
1. Always centralize shared utility functions
2. Don't duplicate code across modules
3. Use OOP/inheritance or utility modules
4. Write static analysis tests to catch duplication
